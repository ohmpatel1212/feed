Six directions for describing what you want to read.

The current product is a chat-driven copilot. These mockups range from refining that idea to abandoning it entirely — sliders, swipes, briefs, dashboards, canvases. Each cell pairs the desktop frame with its mobile sibling so you can judge the tradeoff at both sizes.

01 / Refined Chat

Conversational Copilot

Tradeoff: Lowest cognitive floor — anyone can describe a feed in plain English. But every iteration costs an LLM round-trip and the user can never see the model of their taste they’re building.

02 / No Chat

Direct Tuner

Tradeoff: The user sees the whole model of their taste at a glance — chips, sliders, live count — and every tweak is instant and cheap (no LLM call). But cold-start is harder, and ambiguous vibes like “less doomscrolly” don’t map cleanly to controls.

no chat direct manipulation high density precise cold-start

03 / Tinder for taste

Swipe Studio

Tradeoff: Zero-friction onboarding — the user never has to articulate what they want. The model learns from gestures. But the user can’t see why they’re being shown what they’re shown, and course-correcting a wrong impression takes many swipes.

gestural implicit signal playful tone single-card opaque to user

04 / Editorial

Editorial Briefing

Tradeoff: The product feels like commissioning a magazine, not configuring a filter. Calm, contemplative, makes serious topics feel serious. But the tone discourages quick iteration — users who want to fiddle will feel they’re being held at arm’s length.

literary serif single brief low density slow iteration

05 / Power tool

Dashboard / Studio

Tradeoff: Maximum information per pixel and full keyboard control — scores visible, match reasons in the inspector, ⌘K for everything. Power users get a Linear-grade tool. But the dark-mode density terrifies first-time visitors, and the “why am I here” isn’t obvious.

keyboard-driven dark very high density power user intimidating

spatial node graph FigJam-y legible model poor on mobile
