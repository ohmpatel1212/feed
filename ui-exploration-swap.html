Swap mechanism · 3-column layout

Four ways to switch Chat and Tune in the right column.

Feeds list stays on the left; posts stay in the middle (the destination). The right column toggles between the two ways you talk to the system — conversational and direct. Each variant shows the same content, only the swap UI differs.

Feeds → Posts · fixed → Chat ↔ Tune
01 / Segmented toggle

Segmented Toggle

Tradeoff: The boring answer that’s probably right. Two equal-weight buttons at the top of the column, iOS-style. Discoverable, unmistakable, never out of place. The cost: it’s a generic affordance — it doesn’t hint at the deeper relationship between Chat and Tune.

familiar balanced weight no surprises no extra signal

02 / Tabs with status

Tabs · with status

Tradeoff: Same idea as #01 but each tab carries information — Chat shows unread count, Tune shows a dot when there are unsaved edits. You learn something about the other side without switching. Cost: more visual noise, and counts can become anxiety-inducing.

tabs status-bearing browser-like visual noise

03 / Mode header + swap

Single Mode + Swap

Tradeoff: Only the current mode is named, in a calm serif header. A subtle pill on the right hints at the other mode plus a keyboard shortcut. Quietest UI — no toggles competing for attention. Cost: the inactive mode is one click further away and slightly less discoverable.

quiet UI single name keyboardable less discoverable

04 / Stacked spines

Stacked Spines

Tradeoff: Both modes are always present as compact headers; one is expanded, the other a thin spine you can read at a glance (a few preview chips for Tune; an unread badge for Chat). Click the spine to expand it; the previous mode collapses. Cost: it’s more elaborate than a toggle and the collapsed preview can feel cramped.

always-present preview spines accordion more complex
